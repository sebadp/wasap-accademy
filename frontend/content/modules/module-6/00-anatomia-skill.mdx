## Qué es un Skill

Un **skill** en WasAP es un paquete que le da al LLM la capacidad de **hacer cosas** en el mundo real: consultar el clima, hacer cálculos, buscar en la web, guardar notas. Sin skills, el LLM solo puede generar texto.

<Callout variant="info" title="Tool calling">
Los skills implementan el patrón de **tool calling**: el LLM decide qué herramienta usar, con qué argumentos, y WasAP la ejecuta y le devuelve el resultado.
</Callout>

## Estructura de un Skill

Cada skill tiene dos partes: una **definición** (SKILL.md) y un **handler** (Python):

<FileTree>
  skills/
  ├── calculator/
  │   └── SKILL.md          # Definición: nombre, tools, schemas
  ├── datetime/
  │   └── SKILL.md
  ├── weather/
  │   └── SKILL.md
  └── notes/
      └── SKILL.md
  app/skills/tools/
  ├── calculator_tools.py   # Handler: lógica Python
  ├── datetime_tools.py
  ├── weather_tools.py
  └── notes_tools.py
</FileTree>

## SKILL.md: La Definición

```markdown
---
name: calculator
description: Herramientas matemáticas y de cálculo
version: 1.0.0
tools:
  - name: calculate
    description: Evalúa una expresión matemática de forma segura
    parameters:
      expression:
        type: string
        description: La expresión a evaluar (ej: "sqrt(144) + 2**3")
        required: true
---

## Calculator Skill

Evaluación segura de expresiones matemáticas usando AST.
Soporta: +, -, *, /, **, sqrt, sin, cos, log, abs, round, min, max.
```

## Modelos de Datos

```python
# app/skills/models.py
class ToolDefinition(BaseModel):
    name: str
    description: str
    parameters: dict  # JSON Schema

class ToolCall(BaseModel):
    name: str
    arguments: dict

class ToolResult(BaseModel):
    tool_name: str
    content: str
    success: bool = True
```

<Concept title="JSON Schema para parámetros">
Los parámetros se definen con JSON Schema. Esto permite al LLM generar los argumentos correctos automáticamente. Ollama convierte estos schemas al formato que el modelo entiende.
</Concept>
