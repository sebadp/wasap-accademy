## Qué es un Embedding

Un **embedding** es una representación numérica del significado de un texto. Convierte palabras y frases en vectores (listas de números) donde textos con significado similar están **cerca** en el espacio vectorial, y textos con significado diferente están **lejos**.

<Concept title="Vector = Lista de números">
"Python es un lenguaje de programación" se convierte en `[0.12, -0.45, 0.78, ..., 0.33]` (768 números). Textos sobre programación tendrán vectores parecidos entre sí. Textos sobre cocina tendrán vectores lejanos. Esto permite **buscar por significado** en vez de por palabras exactas.
</Concept>

A diferencia de una búsqueda por keywords (`LIKE '%python%'`), los embeddings entienden que "programador backend" y "Python developer" hablan de lo mismo aunque no compartan palabras.

## nomic-embed-text via Ollama

LocalForge usa `nomic-embed-text` — un modelo de embeddings que corre **localmente** via Ollama. No se envían datos a ningún servicio externo:

```python
# app/llm/client.py — método embed de OllamaClient
class OllamaClient:
    async def embed(
        self,
        texts: list[str],
        model: str | None = None,
    ) -> list[list[float]]:
        """Generate embeddings for a batch of texts via POST /api/embed."""
        url = f"{self._base_url}/api/embed"
        use_model = model or self._model
        payload = {"model": use_model, "input": texts}
        resp = await self._http.post(url, json=payload)
        resp.raise_for_status()
        data = resp.json()
        return data["embeddings"]  # list of 768-float vectors
```

<Callout variant="info" title="API batch: input es una lista">
El método recibe `texts: list[str]` — una lista de textos. Ollama genera todos los embeddings en una sola llamada HTTP. Esto es clave para el backfill eficiente que veremos en lecciones posteriores.
</Callout>

La configuración del modelo vive en `app/config.py`:

```python
# app/config.py — Settings relevantes
class Settings(BaseSettings):
    embedding_model: str = "nomic-embed-text"
    embedding_dimensions: int = 768
    semantic_search_enabled: bool = True
    semantic_search_top_k: int = 10
    memory_similarity_threshold: float = 1.0  # L2 distance threshold
```

<Callout variant="tip" title="768 dimensiones">
Cada embedding tiene exactamente 768 números (floats). Cada dimensión captura un aspecto diferente del significado. Más dimensiones capturan más matices, pero consumen más memoria y cómputo. 768 es un buen balance para un modelo local.
</Callout>

## Distancia L2 (Euclidiana)

Para medir qué tan similares son dos textos, se calcula la **distancia L2** (euclidiana) entre sus vectores:

```python
import math

def l2_distance(a: list[float], b: list[float]) -> float:
    """Distancia euclidiana entre dos vectores."""
    return math.sqrt(sum((x - y) ** 2 for x, y in zip(a, b)))

# Textos similares → distancia pequeña (~0.3-0.7)
d1 = l2_distance(embed("Python dev"), embed("programador Python"))  # ~0.5

# Textos diferentes → distancia grande (~1.2-2.0)
d2 = l2_distance(embed("Python dev"), embed("receta de torta"))    # ~1.8
```

<Concept title="Threshold de similitud">
LocalForge usa `memory_similarity_threshold = 1.0` como umbral. Si la distancia L2 entre el query y una memoria es mayor que 1.0, se considera que no son suficientemente similares. Valores menores (0.5) son más estrictos; valores mayores (1.5) son más permisivos pero pueden traer ruido.
</Concept>

En la práctica, sqlite-vec (la extensión de SQLite que veremos en la siguiente lección) calcula esta distancia de forma optimizada internamente. No necesitás calcularla manualmente.

## Para Qué los Usa LocalForge

| Uso | Cómo | Ejemplo |
|-----|------|---------|
| **Buscar memorias relevantes** | Embed la pregunta del usuario y buscar memorias con vectores cercanos | "¿Cómo iba mi API?" encuentra "Trabaja en API v2 con FastAPI" |
| **Buscar notas** | Embed el query y encontrar notas con contenido similar | "Deploy" encuentra nota titulada "Producción con Docker Compose" |
| **Buscar notas de proyecto** | Embed el query y buscar documentos del proyecto activo | "Testing" encuentra "Plan de tests para API v2" |
| **RAG** | Inyectar contexto relevante al prompt del LLM antes de generar | El LLM responde con información personalizada |
| **Tool discovery** | Embed la descripción de tools para selección semántica | "Quiero crear una nota" matchea con `save_note` |

<Callout variant="warning" title="Embeddings ≠ comprensión">
Los embeddings capturan similitud semántica a nivel de texto, no razonamiento profundo. "La empresa perdió dinero" y "La empresa ganó dinero" pueden tener vectores cercanos porque comparten contexto. No uses embeddings como único criterio para decisiones críticas.
</Callout>

## Límite de Caracteres

nomic-embed-text tiene un contexto de 8192 tokens. Para evitar truncamiento silencioso, LocalForge limita el texto a 6000 caracteres antes de generar el embedding:

```python
# app/embeddings/indexer.py
_MAX_EMBED_CHARS = 6_000  # Conservative limit for mixed-language text

# Uso: content[:_MAX_EMBED_CHARS] antes de llamar a ollama_client.embed()
```
