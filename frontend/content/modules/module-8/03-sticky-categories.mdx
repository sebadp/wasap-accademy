## El Problema de Continuidad

El usuario está trabajando con proyectos y dice "¿Cómo va?". Sin contexto, classify_intent retorna `["none"]` — sin herramientas. Pero el usuario espera una respuesta sobre sus proyectos.

<Callout variant="warning" title="Pérdida de contexto">
Cada clasificación es independiente. El LLM no recuerda que el turno anterior fue sobre proyectos. Las sticky categories resuelven esto.
</Callout>

## Cómo Funcionan

```python
# Phase C en router.py
categories = await classify_intent(messages, context)

if categories == ["none"]:
    # Intentar rescatar con sticky categories
    sticky = await repository.get_sticky_categories(phone_number)
    if sticky:
        # Re-clasificar con contexto de categorías previas
        categories = await classify_intent(
            messages,
            context=f"Previous categories: {sticky}",
        )

# Guardar las categorías actuales para el próximo turno
if categories != ["none"]:
    await repository.save_sticky_categories(phone_number, categories)
```

## Flujo Visual

```
Turno 1: "Creá un proyecto llamado API v2"
  classify → ["projects"]
  save_sticky(["projects"])
  → tools: create_project, list_projects, ...

Turno 2: "Agregá 3 tareas"
  classify → ["projects"]  (keywords claros)
  save_sticky(["projects"])
  → tools: add_task, ...

Turno 3: "¿Cómo va?"
  classify → ["none"]  (ambiguo)
  get_sticky() → ["projects"]
  re-classify con contexto → ["projects"]
  → tools: project_progress, get_project, ...

Turno 4: "Gracias, ahora contame un chiste"
  classify → ["none"]  (chat casual)
  get_sticky() → ["projects"]
  re-classify → ["none"]  (sigue siendo chat)
  → No se actualizan sticky (categories == none)
  → plain chat, sin tools
```

<Concept title="Las stickies NO se sobreescriben con 'none'">
Si la re-clasificación con contexto sigue dando `["none"]`, las sticky categories se mantienen. Solo se actualizan cuando hay categorías concretas.
</Concept>

## Almacenamiento

Las sticky categories se guardan en `conversation_state` (key-value por usuario):

```sql
INSERT OR REPLACE INTO conversation_state
    (phone_number, key, value)
VALUES
    ('+5491100000', 'sticky_categories', '["projects"]');
```
