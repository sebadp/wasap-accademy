## Como Funciona la Clasificacion

`classify_intent()` es una llamada LLM **rapida**: sin tools, sin chain-of-thought, solo texto. Le pedimos al modelo que clasifique el mensaje del usuario en categorias predefinidas.

<Callout variant="info" title="Simple por diseno">
La clasificacion usa `think=False` y no requiere JSON output. El LLM solo retorna nombres de categorias separados por comas. Esto la hace rapida (~200ms) y robusta contra errores de formato.
</Callout>

## La Firma Real

```python
async def classify_intent(
    user_message: str,
    ollama_client: OllamaClient,
    recent_messages: list[ChatMessage] | None = None,
    sticky_categories: list[str] | None = None,
    repository: object | None = None,
) -> list[str]:
```

Nota los parametros opcionales: `recent_messages` provee contexto conversacional, `sticky_categories` es el fallback de continuidad, y `repository` permite cargar prompts versionados desde la base de datos.

## El Prompt del Clasificador

El prompt esta definido como template en `app/skills/router.py` con ejemplos por categoria:

```python
_CLASSIFIER_PROMPT_TEMPLATE = (
    "Classify this message into tool categories. "
    'Reply with ONLY category names separated by commas, or "none".\n'
    "Categories: {categories}, none\n\n"
    "Examples:\n"
    # time
    '"what time is it" -> time\n'
    '"recuerdame en 5 minutos" -> time\n'
    '"set a reminder for tomorrow at 3pm" -> time\n'
    # math
    '"15% of 230" -> math\n'
    # weather
    '"clima en Buenos Aires" -> weather\n'
    # search
    '"busca informacion sobre Python 3.13" -> search\n'
    # notes
    '"remember that I like coffee" -> notes\n'
    # selfcode
    '"cual es tu version?" -> selfcode\n'
    '"busca en el codigo donde se define classify_intent" -> selfcode\n'
    # multi-category
    '"que hora es y como esta el clima?" -> time, weather\n'
    '"calcula 15% de 230 y busca que impuesto aplica" -> math, search\n\n'
    "{recent_context}"
    "Message to classify: {user_message}"
)
```

<Concept title="Few-shot prompting bilingue">
El prompt incluye ejemplos en ingles y espanol porque LocalForge maneja ambos idiomas. Los ejemplos cubren casos simples (una categoria) y multi-categoria. El formato `"mensaje" -> categoria` es mas facil de imitar para el LLM que un JSON schema.
</Concept>

## Contexto Conversacional

Para clasificar mensajes ambiguos como "Ambos" o "Los de los ultimos meses", se incluye el contexto reciente:

```python
recent_context = ""
if recent_messages:
    context_lines = []
    for msg in recent_messages[-6:]:  # Ultimos 3 turnos (6 msgs)
        if msg.role not in ("user", "assistant"):
            continue
        role_label = "User" if msg.role == "user" else "Assistant"
        content_preview = msg.content[:200].replace("\n", " ")
        context_lines.append(f"{role_label}: {content_preview}")
    if context_lines:
        recent_context = (
            "Recent conversation (for context only):\n"
            + "\n".join(context_lines) + "\n\n"
        )
```

<Callout variant="tip" title="Solo 6 mensajes, 200 chars cada uno">
El contexto se limita a los ultimos 3 turnos (user + assistant = 6 mensajes) con un preview de 200 caracteres. Esto mantiene el prompt clasificador ligero (~500 tokens) y rapido.
</Callout>

## El Caso Especial: ["none"]

Cuando el usuario hace chat casual, no necesita herramientas. Pero hay matices:

```python
if raw == "none":
    # Fast-path override: si hay URL, forzar fetch
    if has_url and "search" in TOOL_CATEGORIES:
        return ["search"]

    # Sticky fallback: si el usuario esta en una conversacion con tools
    if sticky_categories:
        return sticky_categories

    return ["none"]
```

<Callout variant="warning" title="URL detection overrides 'none'">
Si el mensaje contiene una URL (detectada con regex), se fuerza la categoria `search` aunque el clasificador diga "none". Esto resuelve el caso comun de "analizame esta pagina: https://..." donde el LLM a veces no detecta que necesita herramientas.
</Callout>

## Parsing de la Respuesta

La respuesta del LLM es texto plano que se parsea:

```python
# Parse comma-separated categories, keep only valid ones
valid = set(TOOL_CATEGORIES.keys())
categories = [c.strip() for c in raw.split(",") if c.strip() in valid]

# Si hay URL y no se detecto search, forzarla
if has_url and "search" not in categories:
    categories.append("search")

# Fallback si no quedo nada valido
if not categories:
    return DEFAULT_CATEGORIES
```

## Fallback: DEFAULT_CATEGORIES

Si la clasificacion falla (output invalido, timeout, excepcion), LocalForge usa un set seguro:

```python
DEFAULT_CATEGORIES = ["time", "math", "weather", "search"]
```

<Concept title="Fail-safe design">
Las categorias default cubren los casos mas comunes (hora, calculo, clima, busqueda). El usuario puede que no obtenga la herramienta exacta, pero no se queda sin herramientas. El fallback es un `try/except` que atrapa *cualquier* excepcion.
</Concept>

## Prompts Versionados (Prompt Engineering)

Si hay un repository disponible, el clasificador puede usar prompts gestionados desde la base de datos:

```python
if repository is not None:
    try:
        from app.eval.prompt_manager import get_active_prompt
        classifier_template = await get_active_prompt("classifier", repository)
    except Exception:
        classifier_template = _CLASSIFIER_PROMPT_TEMPLATE
else:
    classifier_template = _CLASSIFIER_PROMPT_TEMPLATE
```

Esto permite iterar sobre el prompt del clasificador sin deploys -- se cambia en la DB y el siguiente mensaje ya usa la nueva version.

## Tracing del Clasificador

Cada clasificacion genera un span en Langfuse:

```python
trace = get_current_trace()
if trace:
    async with trace.span("llm:classify_intent", kind="generation") as span:
        span.set_input({"user_message": user_message[:200]})
        response = await ollama_client.chat_with_tools(
            messages, tools=None, think=False
        )
        span.set_metadata({
            "gen_ai.usage.input_tokens": response.input_tokens,
            "gen_ai.usage.output_tokens": response.output_tokens,
        })
        raw = response.content.strip().lower()
        span.set_output({"raw": raw})
```
