## Tu primera llamada a Claude

```python title="primer_ejemplo.py"
import anthropic

client = anthropic.Anthropic(api_key="sk-ant-...")

message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "¿Cuál es la capital de Argentina?"}
    ]
)

print(message.content[0].text)
# → "La capital de Argentina es Buenos Aires."
```

## La versión async (la que usa WasAP)

```python title="app/llm/client.py"
import anthropic
from app.config import Settings


class LLMClient:
    def __init__(self, settings: Settings) -> None:
        self.client = anthropic.AsyncAnthropic(
            api_key=settings.anthropic_api_key,
        )
        self.model = "claude-sonnet-4-6"

    async def complete(
        self,
        messages: list[dict],
        system: str | None = None,
        max_tokens: int = 1024,
        temperature: float = 0.7,
    ) -> str:
        response = await self.client.messages.create(
            model=self.model,
            max_tokens=max_tokens,
            temperature=temperature,
            system=system or "",
            messages=messages,
        )
        return response.content[0].text
```

<Concept term="AsyncAnthropic">
La versión asíncrona del cliente. Es esencial en FastAPI porque todas las operaciones son async. Usar el cliente síncrono en un servidor async bloquearía el event loop.
</Concept>

## La estructura de messages

El parámetro `messages` sigue el formato de OpenAI que se volvió estándar:

```python
messages = [
    {"role": "user",      "content": "¿Cuál es la capital de Francia?"},
    {"role": "assistant", "content": "La capital de Francia es París."},
    {"role": "user",      "content": "¿Y de Italia?"},
]
# Claude entiende el contexto: pregunta sobre Italia después de Francia
```

<Callout variant="info">
El historial de conversación se construye alternando `user` y `assistant`. Claude nunca debe ser el primer mensaje ni haber dos del mismo rol seguidos.
</Callout>
