## El Ensamblaje Final

`_build_context()` en `router.py` es una función **sincrónica** que toma todos los datos ya recolectados por `ConversationContext.build()` y construye el prompt final.

<Callout variant="info" title="Sin DB calls">
_build_context() no hace ninguna query a la base de datos ni llamada a Ollama. Solo ensambla strings. Toda la I/O ya se hizo en las fases anteriores.
</Callout>

## _build_context()

```python
def _build_context(
    system_prompt: str,
    memories: list[tuple[str, float]],
    notes: list[str],
    daily_logs: str,
    projects_summary: str,
    capabilities: list[str],
    summary: str | None,
) -> str:
    builder = ContextBuilder(system_prompt)

    # Memorias (ordenadas por relevancia)
    if memories:
        mem_text = "\n".join(f"- {content}" for content, _ in memories)
        builder.add_section("user_memories", mem_text)

    # Proyectos activos
    if projects_summary:
        builder.add_section("active_projects", projects_summary)

    # Notas relevantes
    if notes:
        notes_text = "\n".join(f"- {note}" for note in notes)
        builder.add_section("relevant_notes", notes_text)

    # Actividad reciente
    if daily_logs:
        builder.add_section("recent_activity", daily_logs)

    # Capabilities (solo tools activos)
    if capabilities:
        cap_text = "Available tools: " + ", ".join(capabilities)
        builder.add_section("capabilities", cap_text)

    # Resumen de conversación anterior
    if summary:
        builder.add_section("conversation_summary", summary)

    return builder.build()
```

## _build_capabilities_for_categories()

Las capabilities solo muestran herramientas de las categorías activas:

```python
def _build_capabilities_for_categories(
    categories: list[str],
) -> list[str]:
    if categories == ["none"]:
        return []  # Sin tools, sin capabilities

    capabilities = []
    for cat in categories:
        tools = TOOL_CATEGORIES.get(cat, [])
        capabilities.extend(tools)
    return capabilities
```

<Concept title="Capabilities filtradas">
Si classify retornó `["weather", "notes"]`, las capabilities solo muestran `get_weather, save_note, list_notes, ...`. No se muestra `calculate` ni `web_search`. Esto reduce el ruido en el prompt.
</Concept>

## El Flujo Completo

```
                    Phase A         Phase B          Phase C          Phase D
Mensaje ──► embed + save ──► search all ──► classify ──► _build_context()
             (paralelo)      (paralelo)    (async task)    │
                                                           ▼
                                                    System prompt
                                                    + XML sections
                                                           │
                                                           ▼
                                                   execute_tool_loop()
                                                    o chat() directo
```
