## La Arquitectura de Datos de WasAP

WasAP usa **más de 50 tablas** organizadas en 13 grupos lógicos. Cada grupo encapsula un subsistema del asistente.

<Callout variant="info" title="Idempotencia">
Todas las tablas se crean con `CREATE TABLE IF NOT EXISTS`. Podés ejecutar `init_db()` múltiples veces sin romper nada. Esto es fundamental para updates sin downtime.
</Callout>

## Tablas Core: conversations y messages

```sql
CREATE TABLE IF NOT EXISTS conversations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    phone_number TEXT UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    conversation_id INTEGER NOT NULL,
    role TEXT NOT NULL CHECK(role IN ('user','assistant','system','tool')),
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT (datetime('now')),
    FOREIGN KEY (conversation_id) REFERENCES conversations(id)
);
```

<Concept title="phone_number como identificador">
En WasAP, cada usuario se identifica por su número de teléfono. Para Telegram, se usa `tg_&lt;chat_id&gt;`. Esto evita una migración de schema al agregar plataformas — zero schema migration.
</Concept>

## Índices para Performance

```sql
-- Buscar memorias activas rápidamente
CREATE INDEX IF NOT EXISTS idx_memories_active ON memories(active, id);

-- Buscar resúmenes por conversación (con tiebreaker por id)
CREATE INDEX IF NOT EXISTS idx_summaries_conversation
    ON summaries(conversation_id, id);

-- Mensajes recientes por conversación
CREATE INDEX IF NOT EXISTS idx_messages_conversation
    ON messages(conversation_id, created_at);
```

<Callout variant="warning" title="ORDER BY id DESC">
SQLite `datetime('now')` puede producir timestamps idénticos en el mismo segundo. Siempre usá `ORDER BY id DESC` como tiebreaker para resultados determinísticos.
</Callout>

## Migration Strategy

WasAP no usa un framework de migrations (como Alembic). En vez de eso:

<Step number={1} title="CREATE TABLE IF NOT EXISTS">
Todas las tablas se crean idempotentemente en `init_db()`.
</Step>

<Step number={2} title="Detección de constraints obsoletos">
Para cambios como agregar un valor a un CHECK constraint, WasAP detecta el constraint viejo vía `sqlite_master` y recrea la tabla.
</Step>

<Step number={3} title="Nuevas columnas con ALTER TABLE">
Columnas nuevas se agregan con `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` (SQLite 3.35+).
</Step>
