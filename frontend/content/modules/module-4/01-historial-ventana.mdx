## El Problema: Contexto Limitado

Los LLMs tienen una **ventana de contexto finita** (qwen3.5:9b tiene ~32K tokens). Si una conversaciГіn tiene 200 mensajes, no podГ©s enviarlos todos. Pero si enviГЎs solo los Гәltimos 5, el modelo pierde contexto importante.

<Callout variant="info" title="La soluciГіn">
Enviar los Гәltimos N mensajes **tal cual** (verbatim) + un **resumen** de todo lo anterior. Lo mejor de ambos mundos: detalle reciente + contexto histГіrico.
</Callout>

## get_windowed_history()

```python
async def get_windowed_history(
    self, phone_number: str, verbatim_count: int = 8
) -> tuple[list[dict], str | None]:
    """Retorna (Гәltimos_mensajes, resumen_de_anteriores)."""
    conv_id = await self._get_conv_id(phone_number)

    # Гҡltimos N mensajes, verbatim
    recent = await self.repository.get_recent_messages(
        conv_id, limit=verbatim_count
    )

    # Resumen existente (de la tabla summaries)
    summary = await self.repository.get_latest_summary(conv_id)

    return recent, summary
```

<Concept title="verbatim_count=8">
WasAP usa 8 mensajes verbatim por defecto. Esto incluye ~4 turnos (user+assistant). Suficiente para mantener el flujo conversacional inmediato, sin ocupar demasiado contexto.
</Concept>

## Zero-Latency: No se genera resumen al leer
