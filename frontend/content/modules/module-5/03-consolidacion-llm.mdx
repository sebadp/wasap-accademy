## El Problema: Memorias Redundantes

Con el tiempo, el sistema acumula memorias duplicadas o contradictorias:

```
- "Trabaja como backend developer"
- "Es desarrollador backend en Python"        <- duplicado semántico
- "Trabaja en empresa fintech"
- "Trabaja en una startup de e-commerce"       <- contradicción
```

<Callout variant="info" title="Consolidación">
El consolidador revisa todas las memorias activas, detecta duplicados y contradicciones, y elimina las versiones obsoletas. Es como un "garbage collector" para memorias.
</Callout>

## Detección de Duplicados: difflib

Antes de usar el LLM (costoso en compute), LocalForge intenta detectar duplicados con **similitud de texto** usando `difflib.SequenceMatcher`:

```python
# app/conversation/summarizer.py
from difflib import SequenceMatcher

DEDUP_THRESHOLD = 0.8

def _is_duplicate(new_fact: str, existing: list[str]) -> bool:
    """Check if a fact is too similar to any existing memory."""
    for existing_fact in existing:
        ratio = SequenceMatcher(
            None, new_fact.lower(), existing_fact.lower()
        ).ratio()
        if ratio > DEDUP_THRESHOLD:
            return True
    return False
```

<Concept title="SequenceMatcher ratio > 0.8">
Un ratio de 0.8 significa que el 80% de las secuencias de texto coinciden. Ejemplos prácticos:

- "Trabaja como backend developer" vs "Es desarrollador backend" -> ~0.6 (no duplicado, distintos)
- "Trabaja en fintech" vs "Trabaja en fintech desde 2024" -> ~0.85 (duplicado, uno extiende al otro)
- "Vive en Buenos Aires" vs "Vive en Buenos Aires" -> 1.0 (idéntico)
</Concept>

Este dedup basado en texto se usa en el flush pre-compactación (lección siguiente) para evitar guardar memorias que ya existen. Pero no detecta contradicciones ni duplicados semánticos complejos. Para eso se necesita el LLM.

## Consolidación por LLM

El archivo `app/memory/consolidator.py` implementa la consolidación completa:

```python
# app/memory/consolidator.py
MIN_MEMORIES = 8

CONSOLIDATE_PROMPT = (
    "Review these user memories. Your job:\n"
    "1. Identify duplicates or near-duplicates -> keep the better one\n"
    "2. Identify contradictions -> keep the most recent one "
    "(higher ID = more recent)\n"
    "3. Do NOT remove anything that isn't clearly duplicate or contradicted\n\n"
    "Current memories (oldest first):\n{memories}\n\n"
    'Return JSON: {{"remove_ids": [id1, id2]}}\n'
    'If nothing to remove: {{"remove_ids": []}}'
)
```

<Callout variant="warning" title="think=False obligatorio">
El consolidador usa `think=False` porque necesita una respuesta JSON estructurada, no razonamiento libre. Con `think=True`, qwen3 genera bloques `<think>...</think>` que rompen el parseo JSON.
</Callout>

## La Función consolidate_memories

```python
async def consolidate_memories(
    repository: Repository,
    ollama_client: OllamaClient,
    memory_file: MemoryFile,
    min_memories: int = MIN_MEMORIES,
) -> int:
    """Consolidate memories by removing duplicates and contradictions.
    Returns the number of memories removed."""
    memories = await repository.list_memories()
    if len(memories) < min_memories:
        return 0  # No vale la pena con pocas memorias

    prompt = consolidate_template.format(
        memories=_format_memories(memories)
    )
    messages = [ChatMessage(role="user", content=prompt)]
    response = await ollama_client.chat(messages, think=False)

    # Parse JSON response (con manejo de code fences)
    text = response.strip()
    if text.startswith("```"):
        lines = text.split("\n")
        lines = [l for l in lines if not l.strip().startswith("```")]
        text = "\n".join(lines)
    data = json.loads(text)

    remove_ids = data.get("remove_ids", [])
```

El formato de memorias incluye IDs para que el LLM pueda referenciarlas:

```python
def _format_memories(memories: list) -> str:
    lines = []
    for m in memories:
        lines.append(f"[{m.id}] {m.content}")
    return "\n".join(lines)
```

Ejemplo de input al LLM:
```
[1] Trabaja como backend developer
[2] Es desarrollador backend en Python
[3] Trabaja en empresa fintech
[4] Trabaja en una startup de e-commerce
```

Ejemplo de respuesta:
```json
{"remove_ids": [1, 3]}
```

## Validación de IDs

Antes de eliminar, se valida que los IDs existan y sean enteros:

```python
# Validate IDs: only remove IDs that actually exist
valid_ids = {m.id for m in memories}
remove_ids = [
    rid for rid in remove_ids
    if isinstance(rid, int) and rid in valid_ids
]

removed_count = 0
for memory_id in remove_ids:
    memory = next((m for m in memories if m.id == memory_id), None)
    if memory:
        success = await repository.remove_memory(memory.content)
        if success:
            removed_count += 1
            # Remove embedding (best-effort)
            try:
                await remove_memory_embedding(memory_id, repository)
            except Exception:
                logger.warning("Failed to delete embedding for %d", memory_id)
```

<Concept title="Limpieza de embeddings">
Cuando se elimina una memoria, también se borra su embedding vectorial. Si esto falla, se loguea un warning pero no se interrumpe. El embedding huérfano no causa problemas: simplemente aparecerá en búsquedas pero sin memoria asociada.
</Concept>

## Sincronización Post-Consolidación

Después de eliminar memorias, se sincroniza MEMORY.md:

```python
# Sync MEMORY.md if any were removed
if removed_count > 0:
    updated_memories = await repository.list_memories()
    await memory_file.sync(updated_memories)

logger.info(
    "Memory consolidation: removed %d of %d memories",
    removed_count, len(memories),
)
```

## El Pipeline Completo

```
flush_to_memory()              consolidate_memories()
  |                              |
  v                              v
Extraer facts          Revisar TODAS las memorias
de mensajes viejos     activas del usuario
  |                              |
  v                              v
Crear nuevas memorias  Detectar duplicados (difflib)
(con dedup local)      Detectar contradicciones (LLM)
  |                              |
  v                              v
embed_memory()         Eliminar memorias obsoletas
(indexar vector)       Borrar embeddings huérfanos
                                 |
                                 v
                       Sync MEMORY.md actualizado
```

<Callout variant="tip" title="Background task">
La consolidación corre como tarea programada después del flush, no en el hot path de respuesta al usuario. Se ejecuta dentro de `maybe_summarize()` del summarizer, solo cuando se agregaron nuevos facts.
</Callout>

## Cuándo se Ejecuta

La consolidación se dispara automáticamente en `maybe_summarize()`:

```python
# app/conversation/summarizer.py — al final de maybe_summarize()
if new_facts_count > 0 and daily_log and memory_file:
    try:
        from app.memory.consolidator import consolidate_memories
        await consolidate_memories(repository, ollama_client, memory_file)
    except Exception:
        logger.exception("Memory consolidation failed")
```

Solo se ejecuta cuando `flush_to_memory` extrajo al menos un fact nuevo, evitando llamadas innecesarias al LLM.
